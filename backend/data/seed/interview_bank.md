# Interview Knowledge Seed

## Amazon SDE
- Online assessment commonly emphasizes arrays, hashing, strings, sliding window, heaps, and greedy reasoning.
- Behavioral interviews should use the Leadership Principles and concise STAR stories.
- System design prompts often focus on scalable services, queueing, availability, and tradeoffs.

## Google
- Technical rounds emphasize data structures, algorithms, graph traversal, dynamic programming, and clean reasoning.
- Interviewers value clarifying questions, complexity analysis, and edge-case testing.

## Microsoft
- Expect problem-solving, object-oriented design, debugging, and collaborative communication.
- HR questions often explore learning mindset, teamwork, conflict, and ownership.

## Infosys and TCS
- Common rounds include aptitude, programming fundamentals, SQL, OOP, DBMS, OS, CN, and HR communication.
- HR questions include relocation, strengths, weaknesses, project explanation, and career goals.

## HR Question Bank
- Tell me about yourself.
- Describe a time you handled conflict in a team.
- Why should we hire you?
- What is your biggest weakness, and how are you improving it?

## System Design
- Start with requirements, scale estimates, API design, data model, high-level architecture, bottlenecks, and tradeoffs.
- Mention caching, load balancing, queues, database indexes, consistency, availability, observability, and failure handling.

## Coding Evaluation Rubric
- Correctness: passes standard, edge, and stress cases.
- Complexity: clearly explains time and space costs.
- Communication: explains approach before coding and validates with examples.
